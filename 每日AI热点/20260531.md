## 02 产品发布 / 更新

### 1. Tiny-vLLM 开源：从零理解大模型推理系统

**原文总结：**开发者 Jędrzej Maczan 在 GitHub 开源 Tiny-vLLM 项目。这是一个基于 C++ 与 CUDA 实现的轻量级大模型推理框架，目标并非替代 vLLM，而是帮助开发者学习现代 LLM 推理系统的底层实现原理。项目已经支持 KV Cache、Paged Attention、CUDA Kernel 优化以及 Llama 模型推理等核心能力。

同时，作者还配套提供了大量教学内容，希望通过“从零实现推理引擎”的方式帮助工程师理解 AI Infra 技术栈。

**个人解读：**今年 AI Infra 正在变成新的热门方向。

很多工程师已经会使用 vLLM、TensorRT-LLM 等工具，但真正理解推理框架底层原理的人并不多。Tiny-vLLM 的价值就在于把复杂系统拆解成可学习、可阅读的工程项目。

随着推理优化、KV Cache、Agent 基础设施持续升温，这类项目很可能会成为 AI 工程师学习底层能力的重要入口。

**来源：** GitHub - tiny-vllm

**链接：**https://github.com/jmaczan/tiny-vllm

## 03 行业动态

### 1. xAI 被曝放弃 JAX GPU 训练体系，转向自研 C 语言训练框架

**原文总结：**SemiAnalysis 爆料称，xAI 已逐步放弃基于 JAX 的 GPU 训练方案，转而使用内部代号为 Grok Build 的自研训练框架。据称，xAI 在 JAX 体系下的 GPU 利用率（MFU）长期低于 10%，导致训练效率无法满足需求。消息称，新的训练栈采用 C 语言构建，并已被用于新一代 Grok 模型训练。

报道称，过去两年 NVIDIA 团队曾投入大量资源支持 JAX GPU 生态，但最终未能成为超大规模训练的主流方案。

**个人解读：**如果消息属实，这会是 AI 基础设施领域一个非常重要的信号。

过去几年，行业默认的大模型训练框架主要集中在 PyTorch、JAX 等生态，而 xAI 直接下场自研训练系统，说明头部公司已经开始把训练框架视为核心竞争力。未来算力竞争可能不只是拼 GPU 数量，更是在拼训练效率和系统工程能力。
