**原文：** A coordinated supply chain attack called "TrapDoor" just hit npm, PyPI, and Crates.io simultaneously, 34 malicious packages targeting crypto, AI, and security developers to steal wallets, SSH keys, and cloud credentials. New: attackers are also submitting pull requests to popular open-source repos, injecting manipulated CLAUDE.md and .cursorrules config files. When a developer clones the repo and works with Claude Code or Cursor, the AI agent reads those files as trusted instructions, and could execute malicious commands without the developer realizing it. Using AI assistants as the attack surface is new.

这是一场协调的供应链攻击，同时袭击了 npm、PyPI 和 Crates.io，涉及 34 个恶意包，旨在窃取加密货币、AI 和安全开发者的钱包、SSH 密钥和云凭证。攻击的新手段是向流行开源项目提交 Pull Request，注入被操纵的 CLAUDE.md 和.cursorrules 配置文件。当开发者克隆仓库并使用 Claude Code 或 Cursor 等 AI 助手时，AI 智能体会将这些文件当作可信指令执行，可能在开发者不知情下运行恶意命令。这是首次将 AI 助手作为攻击面。

**总结：** "TrapDoor"供应链攻击开创性地利用 AI 助手作为新型攻击面。攻击者通过向热门开源项目提交包含恶意 CLAUDE.md 和.cursorrules 配置文件的 Pull Request，当开发者使用 Claude Code 或 Cursor 时，AI 会将这些文件视为可信指令并执行恶意命令。此次攻击同时波及 npm、PyPI 和 Crates.io 三大平台，34 个恶意包针对加密、AI 和安全领域开发者，窃取钱包、SSH 密钥和云凭证。这标志着软件供应链安全进入新威胁时代。

### 1. 面向 Codex 的自我优化提示词框架

**原文：** self improvement prompt for codex

Greg Brockman 分享了一个结构化的提示词框架，用于指导 Codex 自动分析其历史记录以识别并固化重复工作流。该框架要求 Codex 回顾会话、Memories 等数据，找出重复、耗时且有明确复用价值的任务。筛选标准包括至少出现两次、输入稳定、可提升效率等。最终，Codex 应以"技能"、子智能体或自动化工具等最小实用形式创建或扩展现有资产，避免冗余。

**总结：** Greg Brockman 分享了 Codex 自我优化提示词框架，指导 AI 自动分析历史会话和记忆数据，识别重复性高、耗时长且有复用价值的工作流。框架要求 Codex 以最小实用形式（技能、子智能体或自动化工具）创建或扩展资产，避免冗余。这一方法论帮助用户建立可持续优化的 AI 协作系统，让工具学习用户的个人模式而非每次重新发明架构。

### 3. Claude Code 自动模式：多任务并行的关键技巧

**原文：** People often ask what my biggest tip is for getting the most out of Claude Code. These days my #1 tip is: use auto mode. Auto mode means no more permission prompts. It is the key building block for multi-clauding: start a session, then while it runs, work on another session in parallel.

Boris Cherny 分享了使用 Claude Code 的最大技巧：使用自动模式。自动模式意味着不再有权限提示，它是实现"多 Claude 并行"的关键构件：启动一个会话，然后在其运行时，并行处理另一个会话。他还解释了自动模式相比"Bypass all permissions"更安全，因为后者可能偶尔执行危险操作（如删除重要文件），而自动模式以更低风险获得更多自主权。

**总结：** Boris Cherny 揭示了高效使用 Claude Code 的核心技巧——启用自动模式。自动模式消除了权限提示，成为"多 Claude 并行"（multi-clauding）的关键基础：用户可以启动一个会话后，在其运行时并行处理其他会话。相比直接绕过所有权限，自动模式更安全，因为它在提供更多自主权的同时降低了模型误操作的风险（如删除重要文件）。这一工作流变革使得开发者可以同时管理多个 AI 会话，大幅提升生产力。
