## 01 模型发布/更新

### 1. Harness-1：把检索子智能体从“会搜”推进到“会组织证据”

**原文总结：** UIUC、UC Berkeley 与 Chroma 联合推出 Harness-1，这是一款基于 gpt-oss-20b 训练的 20B 检索子智能体。它把候选池、证据图、验证记录、重要性标注等“检索过程中的记账工作”放进有状态搜索框架里，由模型专注决定搜什么、保留什么、验证什么以及何时停止。根据原文与论文解读，Harness-1 在 8 个基准上的平均 curated recall 达到 0.730，超过下一个最佳开源检索子智能体 11.4 个百分点，仅落后于 Opus-4.6。模型权重、代码和框架均已公开。

**个人解读：** 这条新闻真正有意思的地方，不只是“又出了一个 20B 模型”，而是它把智能体能力拆成了“推理决策”和“状态管理”两层。对检索型 Agent 来说，这比单纯堆更大上下文更实用，也更像未来企业级智能体的正确方向。

**来源：** MarkTechPost

**链接：** [原文链接](https://www.marktechpost.com/2026/06/06/meet-harness-1-a-20b-retrieval-subagent-trained-with-reinforcement-learning-inside-a-stateful-search-harness-on-gpt-oss-20b/)

## 02 产品发布/更新

### 1. Her：给 Claude Code 会话做“事后复盘”的侦探工具

**原文总结：** Hugging Face 社区文章介绍了 Her，这是一款面向 Claude Code 会话日志的分析工具。用户上传 `.jsonl` 会话后，Her 会用自然语言重建每轮交互，标记部署、配置改动、密钥暴露等高风险动作，并展示 token 消耗、使用过的工具、子智能体、技能和 MCP 服务器。它还支持“Ask Her”追问具体步骤原因，并可做单会话或跨会话项目分析。作者强调，这个工具不调用第三方 AI API，而是在 Hugging Face ZeroGPU 上运行 Nemotron-Mini-4B-Instruct，评估部分保持确定性。

**个人解读：** 大模型编码工具正在从“帮你干活”走向“帮你复盘自己是怎么干的”。Her 这类工具的意义在于，它开始补齐 AI 编程最缺的一块能力：可审计、可解释、可追责。后面这类“AI 过程取证工具”大概率会越来越重要。

**来源：** Hugging Face Blog

### 4. Symbolica 2.0 发布：面向 Python 和 Rust 的可编程符号系统升级

**原文总结：** Symbolica 2.0 正式发布，重点围绕“programmable symbols”展开。新版本强化了 Python 与 Rust 双栈支持，加入更灵活的符号注册和钩子机制，使用户可以自定义符号的化简、求导、打印、级数展开和数值求值行为；同时，评估器新增 JIT 编译、双浮点高精度路径和更多数学函数支持。作者还提到，这一版本在真实场景下带来了从 2 倍到 10000 倍不等的性能提升。

**个人解读：** 这类工具虽然不如大模型新闻热闹，但对科研计算、自动求导、数值优化和工程推导链路很有价值。Symbolica 2.0 的意义，在于它把“高性能符号计算”做得更像一个可编程基础设施，而不只是一个数学工具包。

### 5. OpenAI 披露 Harness Engineering：智能体优先研发是怎么运作的

**原文总结：** OpenAI 在官网文章中介绍了 Harness Engineering 团队如何用 Codex 构建一个几乎完全由 Agent 驱动的软件产品。文章称，这个内部实验项目从空仓库起步，五个月内产出约百万行代码，期间由小团队驱动约 1500 个 PR，并强调“人来引导，Agent 来执行”。团队把知识沉淀进仓库文档，把 `AGENTS.md` 当目录而不是百科全书，再结合浏览器、日志、指标和追踪形成持续反馈回路，让 Codex 能在长任务上稳定工作数小时。

**个人解读：** 这篇文章很值得所有做 AI 编程的人细看。它传递的不是“AI 能替代工程师”，而是工程团队的角色在变化：写代码本身不再是唯一中心，定义环境、组织知识、设计反馈闭环和审查产物，正在变成更关键的能力。

**来源：** OpenAI

**链接：** [原文链接](https://openai.com/index/harness-engineering/)
